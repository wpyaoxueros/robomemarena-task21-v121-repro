# Task21 v126 Parallel Replay: Invalid Runtime Attempts

## Outcome

Seeds 107, 108, and 109 all terminated with `SIGABRT` / exit code 134 during
the first episode. None wrote an episode summary, official score, or video, so
these attempts are invalid and are excluded from every success-rate aggregate.

## Evidence

- Each output contains the complete code/scorer snapshot and a `LIVE_STATUS`
  file marked failed.
- The evaluator was still rolling out when it aborted; VLM logs show ordinary
  autonomous prompts, not an oracle injection.
- The seed107 v126 run manifest was compared with the prior valid v124 seed107
  manifest. Apart from seed/output paths, there was no behavioral parameter
  delta.

## Follow-up

The next experiment is a single-instance seed107 replay with the same frozen
v126 behavior. It changes only process concurrency, so it distinguishes a
runtime/resource abort from a policy or scorer regression.
